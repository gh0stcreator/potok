# πоток — Design Guide v2.0

> Поток вместо хаоса. Уверенность вместо догадок.

Живой документ. Описывает как πоток выглядит, ощущается и говорит.

---

## 1. Принцип

πоток — инструмент для принятия решений, не дашборд для красоты.

**Дизайн должен:**
- Показывать движение, а не статику
- Называть вещи своими именами ("потерял 40% на 2-м шаге", не "Conversion Rate: 60%")
- Помогать принять решение прямо сейчас

**Дизайн не должен:**
- Украшать данные
- Прятать проблему за нейтральным цветом
- Требовать онбординга

## 2. Палитра

### 2.1 Примитивная шкала — 6 серых значений

Сырые значения из Figma. Компоненты НЕ ссылаются на них напрямую — только через семантические токены.

```
gray-0  #0B0B0E   near-black
gray-1  #242628
gray-2  #4E5258
gray-3  #6B7079
gray-4  #A9ACB3
gray-5  #FBFBFB   near-white
```

### 2.2 Семантические токены — поверхности

4 уровня глубины. Используются в обеих темах (значения разные, имена одни).

| Токен | Тёмная | Светлая | Для |
|---|---|---|---|
| `--surface-page` | `#0B0B0E` | `#FBFBFB` | Фон страницы |
| `--surface-raised` | `#242628` | `#F0F1F2` | Сайдбар, хедер |
| `--surface-card` | `#3C3E41` | `#E4E5E7` | Карточки, панели |
| `--surface-overlay` | `#4E5258` | `#D5D6D9` | Модалки, поповеры |

**Принцип:** контент всегда «ближе» к пользователю, чем хром. `surface-card` читается как приподнятое над `surface-page`.

### 2.3 Семантические токены — текст

3 уровня. Инвертируются между тёмной и светлой темой.

| Токен | Тёмная | Светлая | Для |
|---|---|---|---|
| `--fg-default` | `#FBFBFB` | `#0B0B0E` | Основной текст, иконки |
| `--fg-muted` | `#A9ACB3` | `#4E5258` | Лейблы, вторичный текст |
| `--fg-subtle` | `#6B7079` | `#6B7079` | Плейсхолдеры, disabled |

### 2.4 Граница

Один токен, производный через прозрачность:

```css
--border:       rgba(169, 172, 179, 0.15)   /* тёмная тема */
--border:       rgba(11, 11, 14, 0.12)       /* светлая тема */
--border-focus: усиленный вариант для фокус-состояний
```

---

### 2.5 Бренд-цвета (источник: Figma)

| Токен | HEX | Назначение |
|---|---|---|
| `--brand` | `#20208B` | Основной синий — идентичность |
| `--brand-hover` | `#2929B8` | Hover, активный |
| `--brand-dim` | `#181870` | Нажатый, фоновый |
| `--accent` | `#FA683B` | Оранжевый — единственный CTA |
| `--accent-hover` | `#FB7D55` | Hover на accent |

**Правило синего:** навигация, активные состояния, primary-кнопки, прогресс.
**Правило оранжевого:** одна главная CTA на экране + критические метрики (точка слива). Не более 10% экрана.

---

### 2.6 Статусные цвета — функциональные

Каждый цвет решает одну задачу. Не использовать декоративно.

| Токен | HEX | Когда |
|---|---|---|
| `--color-success` | `#5FC62F` | Рост, позитивная динамика, +δ |
| `--color-warning` | `#FBBF48` | Приближение к порогу, требует внимания |
| `--color-error` | `#E84064` | Падение, слив, критическая потеря |
| `--color-info` | `#3B82F6` | Нейтральный контекст, AI-инсайт |

Каждый статус имеет три варианта: основной цвет, `*-bg` (тонированный фон), `*-border` (граница).

```css
/* Пример: бейдж с позитивной метрикой */
background: var(--color-success-bg);
border: 1px solid var(--color-success-border);
color: var(--color-success);
```

### 2.7 Визуализация данных

| Токен | HEX | Назначение |
|---|---|---|
| `--data-gain` | `#5FC62F` | Деньги пришли, рост |
| `--data-loss` | `#E84064` | Деньги ушли, отвал |
| `--data-neutral` | `#A9ACB3` | Базовый поток |
| `--data-primary` | `#2929B8` | Основная метрика |
| `--data-compare` | `#FA683B` | Сравнение, второй ряд |

**Градиент потока** (здоровье воронки):
```
flow-bad → flow-warn → flow-neutral → flow-good
#C4304F    #FBBF48    #6B7079        #5FC62F
```

## 3. Типографика

**Основной:** [Inter](https://rsms.me/inter/) — читаемость на малых размерах, кириллица.
**Моноширинный:** [JetBrains Mono](https://www.jetbrains.com/lp/mono/) — числа, таблицы, данные.

```css
--font-sans: 'Inter', system-ui, sans-serif;
--font-mono: 'JetBrains Mono', ui-monospace, monospace;
```

### Масштаб размеров

| Токен | px | Применение |
|---|---|---|
| `--text-2xs` | 10 | Теги, крошечные метки |
| `--text-xs` | 12 | Caps-лейблы, подписи к чартам |
| `--text-sm` | 14 | Ячейки таблицы, вторичный текст |
| `--text-base` | 16 | Основной текст |
| `--text-lg` | 20 | Заголовок карточки |
| `--text-xl` | 24 | Заголовок секции |
| `--text-2xl` | 30 | Заголовок страницы |
| `--text-3xl` | 36 | Главная метрика |
| `--text-4xl` | 48 | Большой KPI |
| `--text-5xl` | 60 | Splash, hero-число |

### Правила набора

| Элемент | Параметры |
|---|---|
| Заголовки | `weight-bold` 700, `leading-tight` 1.2, `tracking-tight` -0.02em |
| Тело | `weight-regular` 400, `leading-normal` 1.5 |
| KPI числа | `font-mono`, `weight-bold`, `tracking-tight` |
| Лейблы | `text-xs`, `weight-medium`, `tracking-caps` 0.07em, `text-transform: uppercase` |

## 4. Тон и язык

πоток говорит как умный коллега, не как аналитическая система.

**Называй проблему напрямую:**
```
❌ "Conversion rate decreased by 18% in Q4"
✅ "На 2-м шаге вы теряете каждого 5-го клиента"
```

**Используй деньги как единицу:**
```
❌ "CAC increased"
✅ "Каждый новый клиент стоит вам на ₽800 дороже, чем месяц назад"
```

**AI-инсайты — это не украшения:**
```
❌ "AI analysis detected a pattern in your data"
✅ "Пятница = плохой день. 38% отменённых транзакций — в пятницу."
```

## 5. Компоненты

### Кнопки

| Вариант | Токены | Когда |
|---|---|---|
| Primary (CTA) | bg: `--accent`, text: белый | Одна главная CTA на экране |
| Brand | bg: `--brand`, text: белый | Вторичные важные действия |
| Secondary | bg: `--surface-card` + `--border` | Нейтральные действия |
| Ghost | прозрачный, text: `--fg-muted` | Навигация, отмена |
| Danger | `--color-error-bg` + border | Деструктивные действия |

### Структура KPI-карточки

```
[ЛЕЙБЛ — xs, caps, --fg-subtle]
[ЗНАЧЕНИЕ — 4xl, mono, --fg-default]
[ДЕЛЬТА — sm, --color-success/error + стрелка]
[КОНТЕКСТ — xs, --fg-subtle — "vs прошлый месяц"]
```

### Бейдж статуса

```css
background: var(--color-{status}-bg);
border: 1px solid var(--color-{status}-border);
color: var(--color-{status});
border-radius: var(--radius-full);
padding: 3px 10px;
font-size: var(--text-xs);
font-weight: var(--weight-medium);
```

## 6. Do / Don't

### DO

- Показывать δ рядом с числом — всегда контекст
- Использовать `--font-mono` для всех финансовых чисел
- Делать главное число большим — 36–48px
- Называть период у каждой метрики ("за 30 дней")
- Красный/зелёный — только для данных, не для UI-хрома

### DON'T

- Не ставить 5+ метрик без иерархии
- Не синий и оранжевый рядом без разграничения
- Не текст меньше 12px в рабочих интерфейсах
- Не красить нейтральные данные зелёным — только реальный рост
- Не анимировать данные дольше 350ms
