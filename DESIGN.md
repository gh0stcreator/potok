# πоток — Design Guide v1

> Поток вместо хаоса. Уверенность вместо догадок.

Это живой документ. Он описывает как πоток выглядит, ощущается и общается.

---

## 1. Принцип

πоток — инструмент для принятия решений, не дашборд для красоты.

**Дизайн должен:**
- Показывать движение, а не статику
- Называть вещи своими именами ("потерял 40% на 2-м шаге", не "Conversion Rate: 60%")
- Помогать принять решение прямо сейчас

**Дизайн не должен:**
- Украшать данные
- Прятать проблему за нейтральным цветом
- Требовать онбординга

## 2. Цвета

### 2.1 Бренд-палитра (основа)

| Токен | HEX | Назначение |
|---|---|---|
| `--color-brand-primary` | `#20208B` | Основной синий — идентичность бренда |
| `--color-brand-primary-light` | `#2D2DC0` | Hover, активный |
| `--color-brand-primary-dim` | `#181870` | Нажатый, фоновый |
| `--color-accent` | `#EC4E20` | Оранжевый — CTA, критические точки |
| `--color-accent-light` | `#F5673A` | Hover на акценте |
| `--color-steel` | `#C0D6DF` | Светло-голубой — вторичные элементы |

**Правила использования бренд-синего:**
- Навигация, активные состояния, primary-кнопки
- Заголовки секций (не основного контента)
- Фокус-ринги, индикаторы прогресса

**Правила использования акцентного оранжевого:**
- Главная CTA на экране (не более одной)
- Критические метрики (точка слива, просадка)
- Онбординг-шаги, подсказки

**Нельзя:**
- Использовать оба цвета вместе в одном блоке без иерархии
- Красить оранжевым больше 10% экрана
- Синий на чёрном без достаточного контраста (использовать `--color-brand-primary-light`)

---

### 2.2 Тёмная тема — фоновый стек

Тёмная тема строится на пяти уровнях глубины:

```
#000000  bg-base     — страница, самый глубокий фон
#0A0A14  bg-elevated — сайдбары, хедер, фиксированные панели
#12121E  bg-surface  — карточки, панели, контейнеры
#1A1A2E  bg-overlay  — модалки, поповеры, дропдауны
```

**Принцип:** контент всегда ближе к пользователю, чем хром. Карточка `bg-surface` читается как "приближённая" относительно страницы `bg-base`.

---

### 2.3 Утилитарный слой — статусные цвета

Каждый статус решает одну задачу. Не использовать их декоративно.

| Токен | HEX | Когда |
|---|---|---|
| `--color-success` | `#22C55E` | Рост, позитивная динамика, +δ |
| `--color-warning` | `#F59E0B` | Приближение к порогу, требует внимания |
| `--color-error` | `#EF4444` | Падение, слив, критическая потеря |
| `--color-info` | `#3B82F6` | Нейтральный контекст, AI-инсайт |

Каждый статус имеет три варианта: основной, светлый (для иконок/текста), фоновый (для тонированных бейджей).

```css
/* Пример: карточка с позитивной метрикой */
background: var(--color-success-bg);
border: 1px solid var(--color-success-border);
color: var(--color-success-light);
```

---

### 2.4 Визуализация данных — потоки и воронки

Для чартов, санки-диаграмм, воронок:

| Токен | Назначение |
|---|---|
| `--data-positive` (`#22C55E`) | Деньги пришли, конверсия прошла |
| `--data-negative` (`#EF4444`) | Деньги ушли, отвал |
| `--data-neutral` (`#C0D6DF`) | Базовый поток, без оценки |
| `--data-primary` (`#4A4AE0`) | Основная метрика |
| `--data-secondary` (`#EC4E20`) | Сравнение, второй поток |

**Градиент потока** (для визуализации "здоровья" воронки):
```
--flow-loss     → --flow-friction → --flow-neutral → --flow-gain
#C93D12           #F59E0B           #7A9BAA           #22C55E
```

## 3. Типографика

**Основной шрифт:** [Inter](https://rsms.me/inter/)
Чистый, профессиональный, отличная читаемость на малых размерах, поддерживает кириллицу.

**Моноширинный шрифт:** [JetBrains Mono](https://www.jetbrains.com/lp/mono/)
Для чисел, данных, кода. Цифры одинаковой ширины — столбцы выровнены.

```css
font-family: var(--font-sans);   /* Inter */
font-family: var(--font-mono);   /* JetBrains Mono */
```

---

### 3.1 Масштаб размеров

| Имя | Размер | Применение |
|---|---|---|
| `--text-2xs` | 10px | Крошечные бейджи, теги |
| `--text-xs` | 12px | Подписи к чартам, метаданные |
| `--text-sm` | 14px | Ячейки таблицы, вторичный текст |
| `--text-base` | 16px | Основной текст |
| `--text-md` | 18px | Выделенный абзац |
| `--text-lg` | 20px | Заголовок карточки |
| `--text-xl` | 24px | Заголовок секции |
| `--text-2xl` | 30px | Заголовок страницы |
| `--text-3xl` | 36px | Главная метрика |
| `--text-4xl` | 48px | Большой KPI |
| `--text-5xl` | 60px | Сплэш, hero-число |

---

### 3.2 Правила набора

**Заголовки:** `--weight-bold` (700) или `--weight-black` (900), `--leading-tight` (1.2), `--tracking-tight` (-0.02em)

**Тело:** `--weight-regular` (400), `--leading-normal` (1.5), `--tracking-normal`

**Числа/KPI:** `--font-mono`, `--weight-semibold`, `--leading-tight`

**Подписи и лейблы:** `--text-xs`, `--weight-medium`, `--tracking-wide`, `text-transform: uppercase`

```css
/* KPI число — как показывать важную метрику */
.kpi-value {
  font-family: var(--font-mono);
  font-size: var(--text-4xl);
  font-weight: var(--weight-bold);
  line-height: var(--leading-tight);
  letter-spacing: var(--tracking-tight);
  color: var(--text-primary);
}

/* Лейбл над KPI */
.kpi-label {
  font-size: var(--text-xs);
  font-weight: var(--weight-medium);
  letter-spacing: var(--tracking-wide);
  text-transform: uppercase;
  color: var(--text-muted);
}
```

## 4. Текст — тон и язык

πоток говорит как умный коллега, не как аналитическая система.

**Называй проблему напрямую:**
```
❌ "Conversion rate decreased by 18% in Q4"
✅ "На 2-м шаге вы теряете каждого 5-го клиента"
```

**Используй деньги как единицу:**
```
❌ "CAC increased"
✅ "Каждый новый клиент стоит вам на ₽800 дороже, чем месяц назад"
```

**AI-инсайты — это не украшения:**
```
❌ "AI analysis detected a pattern in your data"
✅ "Пятница = плохой день для транзакций. Среди отменённых — 38% в пятницу."
```

## 5. Компоненты — правила

### Кнопки

| Вариант | Фон | Текст | Когда |
|---|---|---|---|
| Primary | `--color-accent` | белый | Одна главная CTA на экране |
| Secondary | `--bg-surface` + border | `--text-primary` | Вторичные действия |
| Ghost | прозрачный | `--text-secondary` | Навигация, ссылки |
| Danger | `--color-error-bg` + border | `--color-error-light` | Деструктивные действия |

### Карточки метрик

Структура карточки KPI:

```
[ЛЕЙБЛ — xs, uppercase, muted]
[ЗНАЧЕНИЕ — 3xl-4xl, mono, primary]
[ДЕЛЬТА — sm, success/error + иконка ↑↓]
[КОНТЕКСТ — xs, muted — "vs прошлый месяц"]
```

### Бейджи статусов

```css
/* Всегда фоновый цвет + граница + соответствующий текст */
.badge-success {
  background: var(--color-success-bg);
  border: 1px solid var(--color-success-border);
  color: var(--color-success-light);
  border-radius: var(--radius-full);
  padding: var(--space-1) var(--space-2);
  font-size: var(--text-xs);
  font-weight: var(--weight-medium);
}
```

## 6. Иконографика

- Использовать **Lucide** или **Heroicons** (outline стиль)
- Размер: 16px (inline), 20px (UI-элементы), 24px (заголовки)
- Цвет иконки = цвет соседнего текста
- Иконки статусов используют `--color-{status}-light`

---

## 7. Отступы и сетка

### Базовый модуль: 4px

Все отступы — кратны 4px. Предпочитаемые значения: 4, 8, 12, 16, 24, 32, 48, 64.

### Сетка страницы

```
Мобайл:   4-колонки, gap 16px, padding 16px
Таблет:   8-колонок,  gap 20px, padding 24px
Десктоп: 12-колонок, gap 24px, padding 32px, max-width 1280px
```

### Внутренние отступы компонентов

```
Кнопки:      py-2 px-4 (8/16px)
Карточки:    p-4 или p-6 (16/24px)
Секции:      py-8 (32px) вертикально
Страница:    px-6 (24px) горизонтально
```

## 8. Скругления

| Токен | Размер | Для |
|---|---|---|
| `--radius-sm` | 4px | Инпуты, мелкие элементы |
| `--radius-md` | 8px | Кнопки, карточки |
| `--radius-lg` | 12px | Большие карточки, модалки |
| `--radius-xl` | 16px | Hero-блоки |
| `--radius-full` | 9999px | Бейджи, аватары, pill-кнопки |

## 9. Тени

На тёмной теме тени делают слои, а не красоту.

```css
--shadow-sm  /* Приподнятые мелкие элементы — тултипы, бейджи */
--shadow-md  /* Карточки над страницей */
--shadow-lg  /* Поповеры, дропдауны */
--shadow-xl  /* Модальные окна */
```

Фокус-эффект использует `--shadow-brand` или `--shadow-accent` — не shadow, а ring.

## 10. Анимация

Минимально. Только там, где помогает понять что происходит.

| Ситуация | Длительность | Кривая |
|---|---|---|
| Hover-состояния | `--duration-fast` (100ms) | `--ease-default` |
| Открытие панелей | `--duration-normal` (200ms) | `--ease-out` |
| Появление данных | `--duration-slow` (350ms) | `--ease-out` |
| Spring/bounce UI | `--duration-normal` | `--ease-spring` |

**Нельзя:** анимировать данные дольше 400ms. Пользователь хочет видеть цифру, а не анимацию.

## 11. Do / Don't

### DO

- Показывать дельту рядом с числом — всегда контекст
- Использовать красный/зелёный только для данных, не для UI-элементов
- Делать главное число большим — 36-48px
- Называть период у каждой метрики ("за 30 дней", "вчера")
- Использовать `--font-mono` для всех финансовых чисел

### DON'T

- Не ставить 5+ метрик без иерархии — одна главная, остальные поддержка
- Не использовать синий и оранжевый рядом без разграничения
- Не делать текст меньше 12px в рабочих интерфейсах
- Не красить нейтральные данные в зелёный — только реальный рост
- Не добавлять анимацию в таблицы и списки
