# πоток — Design Guide v3.0

> Поток вместо хаоса. Уверенность вместо догадок.

Живой документ. Описывает как πоток выглядит, ощущается и говорит.
Источник токенов: `design-tokens.css` v3.0. Figma: `rYGGvsw5L78w5B6osxk9t9`.

---

## 1. Принцип

πоток — инструмент для принятия решений, не дашборд для красоты.

**Дизайн должен:**
- Показывать движение, а не статику
- Называть вещи своими именами ("потерял 40% на 2-м шаге", не "Conversion Rate: 60%")
- Помогать принять решение прямо сейчас

**Дизайн не должен:**
- Украшать данные
- Прятать проблему за нейтральным цветом
- Требовать онбординга

## 2. Палитра

### 2.1 Серая шкала — 6 значений (нейтральная)

Примитивы. Компоненты НЕ используют их напрямую — только через семантические токены.

```
gray-0  #0A0A0A   near-black
gray-1  #171717
gray-2  #262626
gray-3  #666666
gray-4  #A0A0A0
gray-5  #FAFAFA   near-white
```

### 2.2 Поверхности — 4 уровня

| Токен | Тёмная | Светлая | Для |
|---|---|---|---|
| `--surface-page` | `#0A0A0A` | `#FFFFFF` | Фон страницы |
| `--surface-raised` | `#171717` | `#F5F5F5` | Сайдбар, хедер, панели |
| `--surface-card` | `#1C212E` | `#EBEBEB` | Карточки, строки, контейнеры |
| `--surface-overlay` | `#262626` | `#DEDEDE` | Модалки, дропдауны, тултипы |

> Карточка `#1C212E` имеет лёгкую синюю нотку — не нейтральный серый. Это осознанный выбор: создаёт ощущение глубины без агрессивного контраста.

### 2.3 Текст — 3 уровня

| Токен | Тёмная | Светлая | Для |
|---|---|---|---|
| `--fg-default` | `#FAFAFA` | `#0A0A0A` | Основной текст, иконки |
| `--fg-muted` | `#A0A0A0` | `#555555` | Лейблы, второстепенный текст |
| `--fg-subtle` | `#666666` | `#999999` | Плейсхолдеры, disabled |

### 2.4 Граница

```css
--border:       rgba(255, 255, 255, 0.08)   /* тёмная тема */
--border:       rgba(0, 0, 0, 0.09)         /* светлая тема */
--border-focus: усиленный вариант для фокус-состояний
```

---

### 2.5 Бренд-цвета

Три синих. Разные роли — не взаимозаменяемые:

| Токен | HEX | Назначение |
|---|---|---|
| `--brand` | `#20208B` | Логотип, идентичность |
| `--brand-hover` | `#2929B8` | Hover на brand-элементах |
| `--action` | `#447DF7` | **Интерактивный синий** — всё что нажимается: кнопки, ссылки, фокус-кольца, выделенная строка |
| `--action-hover` | `#5B8FF8` | Hover на action-элементах |

**Правило:** `--brand` — только для брендинга (логотип, заглушки). `--action` — для любого UI-взаимодействия. Это разделение позволяет менять один без влияния на другой.

| Токен | HEX | Назначение |
|---|---|---|
| `--accent` | `#FA683B` | Оранжевый — единственный CTA-акцент |
| `--label-indigo` | `#6466E9` | Лейблы, теги, категории |

---

### 2.6 Статусные цвета

| Токен | HEX | Когда |
|---|---|---|
| `--color-success` | `#66DC7E` | Рост, позитивная динамика |
| `--color-warning` | `#DAB33C` | Приближение к порогу, требует внимания |
| `--color-error` | `#E64341` | Падение, слив, критическая потеря |
| `--color-info` | `#447DF7` | Нейтральный контекст, AI-инсайт |

Каждый статус имеет три варианта: `*`, `*-bg`, `*-border`:

```css
/* Бейдж с позитивной метрикой */
background: var(--color-success-bg);     /* rgba(102,220,126,0.12) */
border: 1px solid var(--color-success-border);
color: var(--color-success);
```

### 2.7 Визуализация данных

| Токен | Назначение |
|---|---|
| `--data-gain` = `#66DC7E` | Деньги пришли, рост |
| `--data-loss` = `#E64341` | Деньги ушли, отвал |
| `--data-primary` = `#447DF7` | Основная метрика на графике |
| `--data-compare` = `#FA683B` | Сравнение, второй ряд |
| `--data-neutral` = `#A0A0A0` | Базовый поток, нет направления |

**Градиент здоровья воронки:**
```
flow-bad   →  flow-warn  →  flow-neutral  →  flow-good
#E64341        #DAB33C        #666666          #66DC7E
```

## 3. Типографика

**Основной шрифт:** [Inter](https://rsms.me/inter/) — читаемость на малых размерах, отличная поддержка кириллицы, открытая лицензия (OFL).

**Моноширинный:** [JetBrains Mono](https://www.jetbrains.com/lp/mono/) — числа в таблицах и KPI. Благодаря равной ширине цифр колонки с числами выравниваются без хаоса.

```css
--font-sans: 'Inter', system-ui, sans-serif;
--font-mono: 'JetBrains Mono', ui-monospace, monospace;
```

Подключение через Google Fonts:
```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link href="https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600;700&family=JetBrains+Mono:wght@400;700&display=swap" rel="stylesheet">
```

---

### 3.1 Ролевая типографика — UI

Конкретные размеры для конкретных мест в интерфейсе:

| Роль | px | Вес | Токен | Где |
|---|---|---|---|---|
| **Page title** | 20 | 600 | `--text-lg` | Заголовок страницы / раздела |
| **Section header** | 14 | 600 | `--text-sm` | Заголовок группы, колонки |
| **Table body** | 13 | 400 | `--text-sm` | Строки таблицы, список задач |
| **Nav item** | 13 | 500 | `--text-sm` | Пункты сайдбара |
| **Column header** | 11 | 500 | `--text-2xs`–`--text-xs` | Шапка таблицы, uppercase |
| **Label / tag** | 11 | 500 | `--text-2xs` | Теги, бейджи статусов |
| **Caption** | 11 | 400 | `--text-2xs` | Подписи к чартам, тултипы |
| **KPI number** | 36–48 | 700 | `--text-3xl`–`--text-4xl` | Главная метрика на карточке |
| **KPI label** | 11 | 500 + uppercase | `--text-2xs` | Подпись к KPI, caps |
| **Button** | 14 | 600 | `--text-sm` | Все кнопки |

> Базовый размер тела — **13px, не 16px**. Это compact-продукт. 16px создаёт ощущение детского приложения.

### 3.2 Правила набора

| Элемент | Параметры |
|---|---|
| Заголовки | `weight-semibold` 600, `leading-tight` 1.2, `tracking-tight` -0.02em |
| Тело / строки | `weight-regular` 400, `leading-normal` 1.5 |
| KPI числа | `font-mono`, `weight-bold` 700, `tracking-tight` |
| Caps-лейблы | `text-2xs`, `weight-medium` 500, `tracking-caps` 0.07em, `uppercase` |
| Моно-числа | `font-mono`, 400 — для выравнивания в таблицах |

## 4. Иконки

**Рекомендация: [Lucide](https://lucide.dev)** — единственная библиотека для всего интерфейса.

**Почему Lucide:**
- Открытая лицензия (ISC, бесплатно навсегда)
- 1400+ иконок, единый стиль (outline, 2px stroke)
- React/Vue/Svelte/SVG пакеты из коробки
- Именно такой стиль иконок в референсном интерфейсе
- Используется в Linear, Vercel, Supabase, Paperclip

**Установка:**
```bash
npm install lucide-react        # React
npm install lucide               # ванильный JS / SVG
```

**Использование в React:**
```tsx
import { ArrowRight, TrendingDown, AlertCircle, ChevronDown } from 'lucide-react'

// Размеры для UI-ролей:
<ArrowRight size={16} />      // inline в строке таблицы
<TrendingDown size={20} />    // иконка в навигации
<AlertCircle size={14} />     // иконка в бейдже/лейбле
```

**Правило размеров:**
| Контекст | Размер |
|---|---|
| Inline в тексте | 14px |
| Строка таблицы / список | 16px |
| Навигация | 18–20px |
| Пустое состояние / иллюстрация | 32–48px |

**Правило цвета:** иконка всегда принимает цвет родительского элемента (`currentColor`). Не задавать цвет иконке отдельно — менять цвет у контейнера.

## 5. Тон и язык

πоток говорит как умный коллега, не как аналитическая система.

**Называй проблему напрямую:**
```
❌ "Conversion rate decreased by 18% in Q4"
✅ "На 2-м шаге вы теряете каждого 5-го клиента"
```

**Используй деньги как единицу:**
```
❌ "CAC increased"
✅ "Каждый новый клиент стоит вам на ₽800 дороже, чем месяц назад"
```

**AI-инсайты — это не украшения:**
```
❌ "AI analysis detected a pattern in your data"
✅ "Пятница = плохой день. 38% отменённых транзакций — в пятницу."
```

## 6. Компоненты

### Кнопки

| Вариант | Токены | Когда |
|---|---|---|
| Primary (CTA) | bg: `--accent`, text: белый | Одна главная CTA на экране |
| Action | bg: `--action`, text: белый | Вторичные важные действия |
| Secondary | bg: `--surface-card` + `--border` | Нейтральные действия |
| Ghost | прозрачный, text: `--fg-muted` | Навигация, отмена |
| Danger | `--color-error-bg` + border | Деструктивные действия |

### KPI-карточка

```
[ЛЕЙБЛ — 11px, caps, medium, --fg-subtle]
[ЗНАЧЕНИЕ — 36–48px, mono, bold, --fg-default]
[ДЕЛЬТА — 13px, --color-success/error + иконка стрелки]
[КОНТЕКСТ — 11px, --fg-subtle — "за 30 дней"]
```

### Бейдж статуса

```css
background: var(--color-{status}-bg);
border: 1px solid var(--color-{status}-border);
color: var(--color-{status});
border-radius: var(--radius-full);
padding: 2px 8px;
font-size: 11px;
font-weight: 500;
```

### Строка таблицы

```
высота: 36px
font-size: 13px
padding: 0 16px
border-bottom: 1px solid var(--border)
hover: background var(--bg-hover)
selected: background var(--bg-selected)
```

## 7. Do / Don't

### DO

- Показывать δ рядом с числом — всегда контекст
- Использовать `--font-mono` для всех финансовых чисел
- Делать главную метрику большой — 36–48px
- Называть период у каждой метрики ("за 30 дней")
- Красный/зелёный — только для данных, не для UI-хрома
- Иконки — всегда Lucide, `currentColor`

### DON'T

- Не ставить 5+ метрик без иерархии
- Не brand синий и accent оранжевый рядом без разграничения
- Не текст меньше 11px в рабочих интерфейсах
- Не красить нейтральные данные зелёным — только реальный рост
- Не анимировать данные дольше 350ms
- Не использовать разные иконочные библиотеки одновременно
